Informações Gerais

O Git Copilot é um assistente de programação, funcionando melhor para sugestão e revisão de código. Foi projetado inicialmente para aumentar a produtividade eliminando blocos de códigos repetitivos em boilerplate. Também é um agente de revisão de código para melhorar a segurança e a qualidade do código para commits, além de conseguir revisar altas escalas de PR's para velocidade de automação da produtividade da equipe.

O Copilot é uma ferramenta em uma LLM privada. Ele opera sob um modelo de assinatura paga e as LLMS que o alimentam rodam na nuvem, o que exige conexão com a internet. Seu diferencial competitivo para empresas é a integração profunda e contextual com o ecossistema github que pode executar tarefas de múltiplas etapas como refatoração de código, correção de bugs e gestão de pull requests

Desvantagens e Limitações

  • Assinatura paga por mês ou anualmente
  • Depêndencia a conexão por internet (para alimentação da LLM)
  • Gama mais limitada a linguagens ou técnicas menos utilizadas
  • Exigência de envio de trechos do código para sugestão e análise contextual
  • Melhor funcionamento em plataformas e IDE's integradas a Microsoft

Embora a Microsoft forneçam fortes garantias contratuais e declarações de que seu código não será usado para treinar os modelos, você precisa estar ciente desse tráfego de dados ao lidar com informações sensíveis ou código proprietário em ambientes não corporativos.

Como a GitHub Copilot Funciona

O Fluco básico de uso pode ser definido em 3 pontos, o uso contínuo, a interação direta pelo Copilot CLI e o code review dentro do próprio GitHub

  • 1. Entrada de dados (código normal)
  • 2. Copilot analisa o código em tempo real
  • 3. Comunicação com os servidores da Microsoft
  • 4. Retornos de sugestões, blocos de código ou comandos de execução

Como a ferramenta trabalha principalmente como um 'sugestor' de melhoria e códigos, ele analisa automaticamente suas entradas de dados através do teclado escrevendo, sugerindo melhorias, correções e refatorações completas. se a opção for via terminal você interage diretamente com o CLI, ou chat de agente(assim como agentes de AI tradicionais). também é possível configurar o Copilot diretamente na plataforma do GitHub

  • 1. Análise nas alterações de PR's, feedback instantâneo sobre bugs em potencial e vulnerabilidades de segurança
  • 2. Resumir as alterações de um PR
  • 3. Responder a comentários de revisão, solicitando esclarecimentos ou sugerindo correções
  • 4. Gerar a descrição de um novo PR com base nos commits

Instalação e Setup

A instalação é integrada aos ambientes de desenvolvimento suportados, através de plugins ou extensões (como no visual studio) e da posterior autenticação na sua conta GitHub com uma assinatura ativa (Individual, Pro ou Enterprise). 

                    1. Acesse o marketplace de extensões da sua IDE (ex: "Extensions" no VS Code).
                    2. Pesquise por "GitHub Copilot" (e "GitHub Copilot Chat" se for o caso).
                    3. Clique em "Instalar" na extensão principal do GitHub Copilot.
                    4. Após a instalação, um prompt solicitará que você faça login e autorize a extensão em sua conta GitHub.
                    5. Confirme que sua conta possui uma assinatura ativa do Copilot.

Para usar o Copilot diretamente na linha de comando, você precisa da ferramenta GitHub CLI instalada. 

                    1. Garanta que o GitHub CLI (gh) esteja instalado em seu sistema operacional.
                    2. Instale a extensão Copilot CLI rodando o comando(exemplo no windowsShell):

                    gh extension install github/gh-copilot

                    3. Autentique-se novamente no GitHub CLI se necessário.
                
                    o GitHub Copilot não suporta execução local dos modelos de IA para as suas funcionalidades
                    principais. O processamento do código e a geração de sugestões sempre ocorrem na nuvem
                    (Microsoft Azure/OpenAI). Portanto, não há um comando para usar a ferramenta sem conexão com a internet.

Workflow Assistido por IA

O desenvolvimento assistido pelo Copilot começa com a descrição da funcionalidade ou da tarefa em linguagem natural, seja através de um comentário no código ou no Copilot Chat. A IA analisa o contexto do projeto, os arquivos adjacentes e o prompt para sugerir uma estrutura, um plano de implementação ou um conjunto de comandos no CLI. O código é então gerado de forma assistida, com o desenvolvedor aceitando ou ajustando as sugestões em tempo real. Se surgirem erros durante a fase de testes local, o Copilot Chat pode ser usado para identificar e corrigir o problema, fornecendo explicações claras sobre a causa. Finalmente, o desenvolvedor pode solicitar que o Copilot gere automaticamente testes de unidade para garantir a qualidade do código e crie a documentação necessária para o novo módulo. Dessa forma, o assistente acompanha o desenvolvedor desde a concepção da funcionalidade até a finalização da pull request.

  1. O desenvolvedor descreve a funcionalidade desejada ou cola um trecho de código.
  2. A IA analisa o contexto e sugere uma estrutura ou plano.
  3. O código é gerado, ajustado ou refatorado de forma assistida.
  4. Erros são identificados e corrigidos com explicações.
  5. Testes e documentação podem ser gerados automaticamente.
  6. O Agente de IA pode resumir o trabalho e gerar um rascunho de Pull Request (PR).

Dessa forma, o aluno demonstra como a ferramenta pode se encaixar no dia a dia de um time de desenvolvimento, acelerando o ciclo de feedback e garantindo a qualidade do código.

Principalmente em equipes de desenvolvimento, para velocidade de produção e análise de PR's o Copilot se destaca pela assistência no autocomplete de código e na ampla gama de conteúdo para tecnologias conhecidas

O Workflow fica mais fluido, permitindo que a empresa foque na lógica de negócio de projeto, e os códigos boilerplate sejam mais rápidos se serem feitos. a cadência do time aumenta, uma vez que a hierarquia pode agilizar os processos de análise de PR's dos subordinados

Exemplo prático em Python

Nesta seção, o exemplo mostra como o Copilot assistiu na **geração completa** de uma função de validação de segurança de senha em Python, incluindo o uso de expressões regulares e a correta **tipagem** (`type hinting`).

exemplo_ferramenta.py 
# Exemplo: pequeno validador de senha gerado/refatorado com ajuda da IA

import re

# O Copilot sugere toda a lógica abaixo após o usuário digitar "def validar_senha("
def validar_senha(senha: str) -> list[str] | str:
    erros = []

    if len(senha) < 8:
        erros.append("A senha deve ter pelo menos 8 caracteres.")
    if not re.search(r"[A-Z]", senha):
        erros.append("Inclua ao menos uma letra maiúscula.")
    if not re.search(r"[a-z]", senha):
        erros.append("Inclua ao menos uma letra minúscula.")
    if not re.search(r"\d", senha):
        erros.append("Inclua ao menos um número.")
    if not re.search(r"[\W_]", senha): # Sugestão de caractere especial
        erros.append("Inclua ao menos um caractere especial.")

    return erros or "Senha válida!"


if __name__ == "__main__":
    exemplo_valido = "SenhaSegura@123"
    exemplo_invalido = "curta"
    
    print("--- Teste Válido ---")
    print(validar_senha(exemplo_valido))
    
    print("--- Teste Inválido ---")
    print(validar_senha(exemplo_invalido))
